What is GitHub Pages? OVERVIEW

GitHub Pages is a free static-site hosting service built directly into GitHub. Push files to a repository → your site goes live automatically.

1
Create a repo named username.github.io
2
Enable Pages in Settings → Pages
3
Push your HTML / Markdown files
4
Site is live at https://username.github.io

Key facts

  • 100 % free for public repos
  • Supports Jekyll natively
  • Custom domains supported
  • HTTPS automatically enabled
  • Builds triggered on every git push

What is Reveal.js? FRAMEWORK

Reveal.js is an open-source HTML presentation framework. Slides are just <section> elements — no PowerPoint needed.

✦ Key features

  • Horizontal & vertical navigation
  • Built-in Markdown support
  • Multiple themes (moon, sky, serif, solarized…)
  • Code highlighting via highlight.js
  • Speaker notes, PDF export
  • Hosted on any static server — including GitHub Pages

✦ Minimum HTML

<div class="reveal">
  <div class="slides">

    <section>Slide 1</section>

    <section>
      <section>Slide 2A</section>
      <section>Slide 2B ↓</section>
    </section>

  </div>
</div>

Quarto Presentations ALTERNATIVE WORKFLOW

Quarto is a next-generation publishing system. Write in .qmd (Markdown + code), and Quarto compiles to a Reveal.js HTML presentation automatically.

Minimal .qmd file

---
title: "My Presentation"
author: Soufiane Soulami
format:
  revealjs:
    theme: moon
    slide-number: true
    transition: slide
---

## Slide 1

Content here...

## Slide 2

More content...

Render & publish

1
Install Quarto from quarto.org
2
Write your .qmd file
3
Run quarto render slides.qmd
4
Output: slides.html → push to GitHub
Quarto handles all Reveal.js boilerplate for you.

Deploying to GitHub Pages STEP-BY-STEP

1
Go to your repo: github.com/soufiane01234/soufiane01234.github.io
2
Click Add file → Create new file. Name it presentation.html
3
Paste the entire HTML file content (provided separately)
4
Scroll down → Commit changes (green button)
5
Wait ~60 seconds → visit soufiane01234.github.io/presentation.html
6
Use ← → arrow keys to navigate and ↓ for vertical slides
No build step needed — GitHub serves the HTML directly. ✓
